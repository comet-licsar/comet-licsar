subset_initialise_corners(frame, lon1, lon2, lat1, lat2, sid, is_volc=False, resol_m=30)[source]

This will initialise a subset given by corner lon/lat-s. The results will be stored in $LiCSAR_procdir/subsets

Parameters:

  • frame (str) – frame ID,

  • lon1 (float, float) – corner longitudes (no need to be sorted)

  • lon2 (float, float) – corner longitudes (no need to be sorted)

  • lat1 (float, float) – corner latitudes (no need to be sorted)

  • lat2 (float, float) – corner latitudes (no need to be sorted)

  • sid (str) – string ID (for volcano, use the volclip id (vid) instead of volcano ID (volcid) to keep consistence!)

  • is_volc (bool) – if true, it will set the output folder $LiCSAR_procdir/subsets/volc

  • resol_m (float) – output resolution in metres to have geocoding table ready in (note, RSLCs are anyway in full res)

subset_initialise_centre_coords(frame, clon, clat, sid, is_volc=False, radius_km=12.5, resol_m=30)[source]

This will initialise a subset given by centre lon/lat and radius in km. The results will be stored in $LiCSAR_procdir/subsets

Parameters:

  • frame (str) – frame ID,

  • clon (float) – centre longitude,

  • clat (float) – centre latitude,

  • sid (str) – string ID (for volcano, use its volcano ID number)

  • is_volc (bool) – if true, it will set the output folder $LiCSAR_procdir/subsets/volc

  • radius_km (float) – radius (half of the diameter) of the subset scene, in km

  • resol_m (float) – output resolution in metres to have geocoding table ready in (note, RSLCs are anyway in full res)

get_master(frame, asfilenames=False, asdate=False, asdatetime=False, metafile=None)[source]

Gets reference epoch of given frame, returns in several ways

Parameters:

  • frame (str) – frame ID

  • asfilenames (bool) – returns as filenames that were used to create this ref. epoch during init

  • asdate (bool) – returns as dt.datetime.date

  • asdatetime (bool) – will include also the acquisition centre time (returns as dt.datetime)

  • metafile (str) – path to metadata file of the frame. if None, it will search in LiCSAR_public

bursts2geopandas(bidtanxs, merge=False, use_s1burst=False)[source]

Gets geopandas layer for a list of burst IDs (in the form of bidtanx, i.e. e.g. 73_IW1_1234

Parameters:

  • bidtanxs (list) – list of burst ids, e.g. [‘73_IW1_1234’]

  • merge (bool) – whether to merge the output into one polygon

  • use_s1burst (bool) – use the official S-1 burst polygons (more accurate, including burst overlaps)

frame2geopandas(frame, brute=False, use_s1burst=False, merge=False)[source]

Gets geopandas layer for a frame

Parameters:

  • frame (str) – frame ID

  • brute (bool) – do not use polygons in LiCSInfo, but regenerate them instead (by quite brute/slow approach)

  • merge (bool) – whether to merge the bursts into one (frame) polygon

  • use_s1burst (bool) – use the official S-1 burst polygons (more accurate, including burst overlaps)

generate_new_frame(bidtanxs, testonly=True, hicode=None)[source]

Main function to generate new frame definition based on selected bursts :param bidtanxs: list of burst IDs in the bidtanx form, e.g. [‘73_IW1_1234’,’73_IW1_2345’,..] :type bidtanxs: list :param testonly: if True, only perform dry run for debugging :type testonly: bool :param hicode: special code for non-standard frame resolution; use ‘H’ for high resolution (1/5 multilook), or ‘M’ for medium, i.e. 56 m - not much used but should work ok :type hicode: str or None

epoch_has_all_frame_bursts(epoch, frame)[source]

Checks if epoch contains all necessary bursts in frame definition. Needs to have the epoch already ingested in LiCSInfo database.

Parameters:

  • epoch (dt.datetime.date, dt.datetime or str) – epoch date (if str, should be as 20191120)

  • frame (str) – frame ID

Returns:

True means Yes, it has all bursts. False means it has missing bursts”

Return type:

boolean
